## Автоматизированная система обработки почты
Система предназначена для классификации входящих обращений, их сортировки по папкам и построению графиков.

## Основные возможности
__Двухуровневая классификация__:

ㅤㅤОсновная версия:ㅤна основе весовых коэффициентов и регулярных выражений.

ㅤㅤС использованием OpenAI API:ㅤИспользование GPT-3.5 Turbo для глубокого анализа писем.

__Устойчивость к данным__: Обработка файлов типа .txt и .emg, игнорирование системных файлов (.DS_Store) и обработка поврежденных/неизвестных форматов (.bin, .jpeg и др.).

__Визуализация__: графическая интерпретация сохраняется в analytics.png

__Логгирование__: логи сохраняются в classification.log

## Принцип работы

Код реализован на ООП с использованием трёх разных классов:

__Mail__: Объектная модель письма. Отвечает за парсинг заголовков и тела письма на русском и английском языке, также проверяет транслит.

__MailClassifier__: Основной классификатор + AI-классификатор:

ㅤㅤОсновной классификатор:ㅤИз письма извлекаются поля From, to, Subject, Body. Для каждой категории задан набор фраз с весами. Текст (тема + тело) проверяется на все фразы, очки суммируются. Выбирается категория с наибольшей суммой.

ㅤㅤAI-классификатор:ㅤклассификация через OpenAI API с проверкой на валидность ответа.

__MailAnalytics__: Модуль расчета времени обработки и построения графиков

### Категории обработки

__Incidents__ – критичные сбои, срочные запросы к поддержке.

__Access__ – запросы на доступы, оформление новых сотрудников.

__Spam__ – фишинг, реклама, подозрительные ссылки, выигрыши.

__Support__ – частые технические проблемы, которые не приводят к критическим проблемам.

__Info__ – дайджесты, плановые работы, приглашения, отчёты мониторинга.

__Documents__ – счета, акты, договоры, инструкции.

__Other__ – всё остальное.
